368 /*
369  * The following three functions give the unsigned absolute value of a signed
370  * integer even for the most negative integer. This is beyond the scope of
371  * the standard abs() function and its family, whose return-values are signed.
372  * In short, we should not use the unary minus operator with signed numbers
373  * unless we are sure that there are no integer overflows. Instead, we rely on
374  * two well-defined operations: (i) signed-to-unsigned conversion and
375  * (ii) unary minus of unsigned operands.
376  *
377  * See also:
378  * https://stackoverflow.com/a/4536188 (Unary minus and signed-to-unsigned conversion)
379  * https://stackoverflow.com/q/8026694 (C: unary minus operator behavior with unsigned operands)
380  * https://stackoverflow.com/q/1610947 (Why does stdlib.h's abs() family of functions return a signed value?)
381  * https://blog.regehr.org/archives/226 (A Guide to Undefined Behavior in C and C++, Part 2)
